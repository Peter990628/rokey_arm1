## 7. ⚠️ 오류/예외처리 목록 (Error Handling)

프로젝트 전 구간(웹 대시보드, DB 연동, 로봇 제어)에서 실제로 대응한 예외 상황들입니다.

### 7-1. 웹 대시보드 (`app.py`)

| 예외 상황 | 발생 위치 | 처리 방안 |
|---|---|---|
| DB API 서버 연결 실패 (네트워크 끊김, 서버 다운) | `get_inventory()`, 처방전 조회 등 | `requests.exceptions.RequestException` 캐치 → `[ERROR]` 로그 남기고 빈 리스트 반환. 화면이 죽지 않고 "데이터 없음"으로만 표시됨 |
| DB 응답에 필수 필드(`medicine_number`, `medicine_name`) 누락 | `get_inventory()` | 해당 항목만 `[WARN]` 로그와 함께 스킵, 나머지 정상 항목은 그대로 표시 |
| DB 응답의 `created_at` 시각 형식이 이상함 | `format_kst()` | `ValueError` 캐치 → `[WARN]` 로그, 원본 문자열 그대로 표시(화면이 죽지 않음) |
| DB가 예상 못한 `status` 값(`WAITING`/`PROCESSING`/`DONE` 외) 반환 | `map_status()` | `[WARN]` 로그 남기고 안전하게 `production`(제조중)으로 처리 |
| 처방전 등록 시 재고 부족 | `create_prescription_remote()` | DB가 부족 항목 목록과 함께 `400` 응답 → 프론트에 그대로 전달, 알림으로 표시 |

### 7-2. 처방전 입력 폼 (`prescription.html`)

| 예외 상황 | 발생 위치 | 처리 방안 |
|---|---|---|
| 같은 약을 여러 행에서 중복 선택 | 약 선택 드롭다운 | 이미 다른 행에서 선택된 약은 목록에서 아예 제외 (선택 자체가 불가능하게 사전 차단) |
| 수량을 0 이하 또는 51개 이상으로 입력 시도 | 수량 `+`/`-`, 직접 입력 | `Math.min(50, Math.max(1, ...))`로 항상 1~50 범위로 강제 클램핑 |
| 등록 가능한 약을 모두 다 추가한 상태에서 "약 추가" 클릭 | `addMedRow()` | 버튼 자동 비활성화 + "모든 약이 추가되었습니다" 안내, 추가 행 생성 자체를 막음 |
| 환자 이름/약 정보 없이 등록 시도 | 폼 제출 핸들러 | 제출 전 검증, `alert`로 안내 후 요청 자체를 보내지 않음 |
| 서버와 통신 실패(네트워크 오류) | `fetch()` 호출부 | `try/catch`로 감싸서 `alert` 표시, 페이지가 멈추지 않음 |

### 7-3. 로봇 제어 노드 (`scraper_packaging_node.py`)

| 예외 상황 | 발생 위치 | 처리 방안 |
|---|---|---|
| 토픽으로 온 메시지가 JSON 형식이 아님 | `_parse()` | `json.JSONDecodeError` 캐치 → `[WARN]` 로그, `None` 반환하여 콜백에서 안전하게 무시 |
| `items` 배열이 비어있는데 "전부 준비됨"으로 잘못 판단될 위험 | `on_task_event()` | `bool(items) and all(...)` 형태로 빈 배열은 명시적으로 제외 (파이썬 `all([])==True` 함정 방지) |
| 이미 다른 처방 처리 중인데 새 픽업 신호가 또 들어옴 | `_start_pickup()` | `self.state` 확인 후 `IDLE`이 아니면 `[WARN]` 로그와 함께 무시 (로봇 팔 하나로 동시 처리 불가하므로) |
| 아직 스크래퍼도 안 든 상태(대기 중 아님)인데 완료 신호가 들어옴 | `_start_pour()` | `self.state`가 `WAITING`이 아니면 무시 |
| **개별 이동/그리퍼 동작 중 하나라도 실패 (로봇 응답 없음, 통신 끊김 등)** | **`_step()`** | **동작 하나하나(`movej`/`movel`/`grip`/`release`/`set_tcp`)를 감싸는 공통 헬퍼. 실패 시 `[단계 실패] 제조기 이동 2/4 (Z +160): ...`처럼 정확히 어느 단계인지 로그로 남기고, 예외를 다시 던져 상위 로직이 상태 복구하게 함** |
| 동작 도중(위 `_step`에서 못 잡은 예외 포함) 최종적으로 실패 | `_start_pickup()`, `_start_pour()` | `try/except/finally`로 감싸서, 실패해도 `state`를 반드시 안전한 값으로 되돌려 다음 처방 처리에 지장 없게 함 |
| 조인트 한계 초과 좌표로 이동 시도 | 로봇 컨트롤러 레벨 (`JNT POS LIMIT ERROR`) | 컨트롤러가 자체적으로 이동을 거부. 실측 좌표를 재티칭하여 한계 이내 값으로 수정 |
| `DSR_ROBOT2` 모듈 import 실패(환경 미설정) | `main()` | `ImportError` 캐치 → 에러 로그 남기고 노드를 안전하게 조기 종료 |
